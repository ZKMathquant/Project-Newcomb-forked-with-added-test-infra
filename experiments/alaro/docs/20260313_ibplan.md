# Plan: Infrabayesian Agent Implementation

## Background

Infrabayesianism generalizes Bayesian inference by replacing a single prior
distribution with a **credal set** — a closed, convex set of probability
distributions. Where a Bayesian agent maximizes expected utility under one
distribution, an infrabayesian agent maximizes **worst-case expected utility**
over the credal set:

    π* = argmax_π  min_{P ∈ C}  E_P[U(π)]

This captures **Knightian uncertainty** (uncertainty about the uncertainty
itself). The "width" of the credal set parameterizes how much ambiguity the
agent tolerates. A singleton credal set recovers standard Bayesianism.

## Why infrabayesianism for this repo

The existing agents (Q-learning, Bayesian, EXP3) treat each environment as a
standard bandit: they estimate per-action reward values and ignore
policy-dependence. In Newcomb-like environments, the reward depends on the
agent's full policy (because the predictor samples from it via `predict()`).
These agents are therefore blind to the core structure of the problem.

An infrabayesian agent addresses this directly. It maintains uncertainty over
the **reward table** (how rewards depend on both the prediction and the action)
and optimizes its **policy** — the full probability distribution returned by
`get_probabilities()` — against the worst case in its credal set. This is
exactly what the Newcomb-like environments demand.

## Does the bandit architecture suffice?

**Yes.** The current architecture already has the right interface:

- `get_probabilities()` returns a full distribution over actions (not just an
  action), so the agent can reason about and optimize over policies.
- `predict(probabilities)` lets the environment observe the agent's policy
  before the action is sampled, enabling policy-dependent rewards.
- `update(probabilities, action, reward)` gives the agent everything it needs
  to refine its credal set.

There are no states in these environments — just repeated interactions with a
predictor — so full RL (with state transitions) is unnecessary. The
infrabayesian agent optimizes over the space of mixed strategies, which for
k=2 actions is a 1D optimization over p ∈ [0,1]. This is computationally
trivial.

## Should the agent handle Newcomb's problem?

**Yes, by construction.** The maximin over the credal set naturally handles
policy-dependent rewards. When the credal set includes hypotheses where the
predictor is accurate, the worst-case analysis forces the agent to account for
the correlation between its policy and the predictor's behavior. With a
sufficiently accurate predictor (and a credal set that reflects this), the
agent will one-box in Newcomb's problem and play the minimax equilibrium
(p=0.5) in Death in Damascus.

The key insight: an infrabayesian agent doesn't need to "know" it's in a
Newcomb-like environment. If its credal set is broad enough to include
policy-dependent hypotheses, it will be robust to them automatically.

## Implementation plan

### Step 1: Credal set representation

Represent the credal set as **interval bounds on the reward table entries**.
For a 2-action problem, the agent maintains:

- `reward_lower[i,j]` — lower bound on reward when predictor predicts i,
  agent plays j
- `reward_upper[i,j]` — upper bound on same

This is an interval model: the credal set is the set of all reward tables T
where `reward_lower[i,j] ≤ T[i,j] ≤ reward_upper[i,j]`.

Initialize with wide bounds (e.g., `[-R_max, +R_max]`) representing near-total
ignorance. As the agent observes rewards, the bounds contract.

### Step 2: Worst-case expected reward computation

Given a candidate policy p = (p_0, p_1, ..., p_{k-1}) and interval bounds on
the reward table, compute:

    V(p) = min_T  Σ_{i,j}  p_i · p_j · T[i,j]
           s.t.  reward_lower[i,j] ≤ T[i,j] ≤ reward_upper[i,j]

Since each T[i,j] appears linearly, the minimum is achieved at a vertex of
the interval constraints. For each (i,j), use `reward_lower[i,j]` if
`p_i · p_j > 0` (the pessimistic choice). This makes the worst-case
computation O(k²) — just pick the lower bound for each cell.

Note: a more nuanced version could use `reward_upper[i,j]` when the
coefficient is negative, but with non-negative probabilities this doesn't
arise. Worth revisiting if we add environments with negative reward
dependencies.

### Step 3: Policy optimization

For k=2 actions, optimize over p ∈ [0,1]:

    p* = argmax_{p ∈ [0,1]}  V(p)

where V(p) is the worst-case expected reward from Step 2. Since V(p) is
piecewise-quadratic in p (quadratic within each region where the same vertex
of the interval bounds is active), this can be solved:

- **Simple approach**: Grid search over p with resolution ~100 points.
  Sufficient for k=2 and fast.
- **Better approach**: Since V(p) is quadratic for fixed bounds, solve
  analytically. For each possible active set of bounds, compute the optimal p
  via calculus, then take the best.

For k>2, this becomes a constrained optimization over the simplex. Use
scipy.optimize or discretize the simplex. Not needed for the current
environments (all Newcomb-like ones require k=2), but worth designing for.

### Step 4: Credal set update

After observing action a and reward r, update the interval bounds.

**Challenge**: The agent doesn't observe the predictor's prediction directly.
It only knows its own policy p and that the predictor sampled from p. So the
observed reward is a mixture:

    r comes from T[i, a] with probability p_i  (where i was the prediction)

**Approach**: Maintain running statistics per action and use them to tighten
bounds. One concrete scheme:

1. Track per-action empirical mean and count: `μ_a`, `n_a`
2. Use confidence-interval-style bounds:
   `reward_lower[i,a] = max(reward_lower[i,a], μ_a - δ(n_a))`
   `reward_upper[i,a] = min(reward_upper[i,a], μ_a + δ(n_a))`
   where `δ(n) = C / √n` for some confidence parameter C.

This is conservative (the bounds are wider than necessary because the agent
can't separate T[0,a] from T[1,a] without observing the prediction). A
smarter version could use the policy p to partially disentangle the mixture,
but the conservative version is a reasonable starting point.

**Alternative**: If exploration phases use near-deterministic policies (like
ExperimentalAgent2 does with `exploration_peak`), the agent can effectively
observe individual entries of the reward table. This could be incorporated as
a mode.

### Step 5: Class structure

```
InfraBayesianAgent(BaseAgent)
    __init__(num_actions, seed, verbose,
             confidence=1.0,    # controls credal set width (higher = wider = more robust)
             reward_range=20.0, # initial bound magnitude
             resolution=101)    # grid search resolution for policy optimization

    reset()
        Initialize reward_lower = -reward_range * ones(k, k)
        Initialize reward_upper = +reward_range * ones(k, k)
        Initialize counts = zeros(k)
        Initialize reward_sums = zeros(k)

    get_probabilities() -> NDArray
        Grid search over policies to find argmax of worst-case expected reward
        Return optimal policy

    update(probabilities, action, reward)
        Update counts[action] += 1
        Update reward_sums[action] += reward
        Recompute empirical mean for action
        Tighten interval bounds using confidence intervals

    dump_state() -> str
        Show current policy, bounds, counts
```

### Step 6: Registration

Add to `ibrl/agents/__init__.py`:
```python
from .infrabayesian import InfraBayesianAgent
```

Add to `ibrl/utils/construction.py` agent_types dict:
```python
"infrabayesian": agents.InfraBayesianAgent
```

Invocable as `"infrabayesian"` or `"infrabayesian:confidence=2.0"` etc.

### Step 7: Testing

Test against all existing environments, focusing on:

1. **bandit** — should converge to the best arm (degenerate case, credal set
   collapses to point estimates, recovers standard bandit behavior)
2. **newcomb** — should converge toward one-boxing (p(action 0) → 1) with
   tight enough credal set
3. **damascus** — should converge toward p ≈ 0.5 (minimax equilibrium)
4. **asymmetric-damascus** — should find the asymmetric equilibrium
5. **coordination** — should converge to the better equilibrium (p(action 0) → 1)
6. **pdbandit** — should adapt to random reward tables
7. **switching** — should track the switch (tests adaptivity)

## Open questions

1. **Credal set contraction rate**: How fast should the bounds shrink? Too fast
   and the agent becomes overconfident (degenerates to Bayesian). Too slow and
   it's too conservative. The `confidence` parameter controls this, but the
   right default needs experimentation.

2. **Mixture disentangling**: The agent observes rewards that are mixtures over
   predictor predictions. Can we use EM-style updates or moment matching to
   better separate the contributions of different reward table entries?

3. **k > 2 actions**: The current Newcomb-like environments all use k=2, so
   policy optimization is 1D. For k>2, we'd need simplex optimization. Not
   urgent but worth designing the interface to support it.

4. **Exploration vs. robustness**: The infrabayesian agent is inherently
   pessimistic (worst-case optimization). Should we add an explicit exploration
   bonus (infra-Bayesian UCB style) to avoid getting stuck? Or does the wide
   initial credal set provide enough implicit exploration?

5. **Non-stationary environments**: The `switching` environment changes the
   reward structure mid-run. Should the credal set re-widen after detecting a
   change? Or does the inherent conservatism of wide bounds handle this?
